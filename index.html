Biggy

A very fast document/relational query tool
with Full LINQ Compliance.

source code can be found at Github

Quick Start

We know you want to get started as fast as possible, so follow these steps to get Biggy into your application.

Installation

We suggest you install Biggy using Nuget. It is the fastest way to get started. You may utilize the visual package manager that is part of Visual Studio or use the following command in your package manager console.

PM> Install-Package Biggy

Awesome! You are ready to write your first Biggy app

Your First Application

Start off by creating a new Console Application in Visual Studio. Then reference Biggy using Nuget, see installation section for help. You should have an empty console application now.

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace ConsoleApplication3
{
    class Program
    {
        static void Main(string[] args)
        {
        }
    }
}

Your next step is to create a storage class. These classes are plain old C-Sharp objects, more commonly referred to as POCOs. Paste the following code into your class into your console application.

class Product {
  public String Sku { get; set; }
  public String Name { get; set; }
  public Decimal Price { get; set; }
  public DateTime CreatedAt { get; set; }

  public Product() {
    this.CreatedAt = DateTime.Now;
  }

  public override bool Equals(object obj) {
    var p1 = (Product)obj;
    return this.Sku == p1.Sku;
  }
}

Finally, let's interact with Biggy. Paste the following code into your project.

static void Main(string[] args)
{
    //add and save to this list as above
    //this will create a Data/products.json file in your project/site root
    var products = new BiggyList<Product>();

    var newProduct = new Product { Sku = "STUFF", Name = "A new product", Price = 120.00m };

    //gets appended immediately in a single line-write to file
    products.Add(newProduct);

    //this won't hit the disk as you're querying in-memory only
    var p = products.FirstOrDefault(x => x.Sku == "STUFF");
    p.Name = "Something Fun";

    //this writes to disk in a single asynchronous flush - so it's fast too
    products.Update(p);

    Console.ReadLine();
}

Hooray! If you place some breakpoints in the application you will notice that we have an IEnumerable of products. You might also notice we were able to utilize LINQ to query the product. In addition, this example is saving the product to disk asynchornously so you don't lose any data. All of this is done for you transparently.

Welcome to Biggy, please continue reading to learn about the other features of the library and please follow us on Github. If you struggled with the sample, see the full source below to help you.
